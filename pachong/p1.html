CentOS 7 安装配置 PPTP VPN服务端

PPTP协议是VPN族中最不安全的协议，要过天朝firewall的话，推荐其它方法：

  • Ubuntu 编译安装 Shadowsocks-libev
  • Ubuntu 16.04 安装配置 Shadowsocks 服务端
  • IPsec/L2TP VPN安装脚本－Ubuntu/Debian
  • CentOS 7 安装配置 SoftEther VPN服务端

CentOS 7安装配置PPTP

#1 安装前检查系统

在安装之前查看系统是否支持PPTP VPN：

1
modprobe ppp-compress-18 && echo success

应该输出”success”，如果不支持尝试安装：

1
yum install kernel-devel

是否开启TUN/TAP：

1
cat /dev/net/tun

应该输出：

1
cat: /dev/net/tun: File descriptor in bad state

是否开启ppp：

1
cat /dev/ppp

应该输出：

1
cat: /dev/ppp: No such device or address

确保满足上面3个条件，如果不满足，表示不能安装PPTP VPN。

#2 安装软件包

pptpd在epel仓库中，首先安装epel仓库：

1
yum install epel-release

安装需要的软件包：

1
yum install ppp pptpd net-tools iptables-services

 #3 配置pptpd

编辑/etc/pptpd.conf：

1
vim /etc/pptpd.conf

写入内容：

1
2
3
4
option /etc/ppp/options.pptpd
logwtmp
localip 10.0.10.1  # 设置pptp虚拟网卡接口IP地址（注意：不是服务器的公共IP）
remoteip 10.0.10.2-254 # 为拨入vpn的用户动态分配的IP地址池

编辑/etc/ppp/options.pptpd：

1
vim /etc/ppp/options.pptpd

写入内容：

1
2

4

name pptpd
 
refuse-pap
refuse-chap
refuse-mschap
require-mschap-v2
require-mppe-128
 
proxyarp
 
lock
nobsdcomp
 
novj
novjccomp
nologfd
 
ms-dns 8.8.8.8  #添加主DNS服务器地址
ms-dns 8.8.4.4  #添加备DNS服务器地址

添加VPN用户和密码：

1
vim /etc/ppp/chap-secrets

写入内容：

1
2
3
4
# Secrets for authentication using CHAP
# client server secret IP addresses
user_test1       pptpd   123456                 *
user_test2       pptpd   123456                 *

*代表所有IP都可以连接本vpn用户。

#4 开启IP转发

1
2
echo net.ipv4.ip_forward = 1 >> /etc/sysctl.conf
sysctl -p   # 使生效

#5 配置防火墙

停用所有防火墙：

1
2
3
4
5
6
systemctl stop firewalld
systemctl disable firewalld  # 如果使用firewallid
service iptables save
service iptables stop
chkconfig iptables off
iptables -F

设置防火墙转发规则：

1
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

注意替换上面的网卡接口名(eth0)，使用ifconfig查看。

设置开机自动执行上面的命令：

1
2
echo "iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE" >> /etc/rc.d/rc.local
chmod +x /etc/rc.d/rc.local

重启pptpd服务：

1
service pptpd restart
